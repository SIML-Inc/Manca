# x402 Settlement (real USDC)

Manca's clearing logic is rail-agnostic: it decides **when** a trade settles; the
settlement rail decides **how the value actually moves**. The `x402` rail moves
USDC over the [x402 protocol](https://x402.org) (HTTP 402 + EIP‚Äë3009 `exact`
scheme, facilitator `/verify` + `/settle`).

## Three modes (safe by default)

| mode | what happens | needs |
|---|---|---|
| **`mock`** (default) | deterministic pseudo tx hash, **fully offline, nothing real moves** | nothing |
| **`testnet`** | real **Base Sepolia** testnet USDC via the x402 facilitator | `viem` + env + payout addresses |
| **`mainnet`** | real **Base** USDC ‚Äî **REAL money** | all of testnet **+** explicit opt‚Äëin |

Check the current mode any time:

```bash
npm run x402:status
```

Ships in **`mock`** ‚Äî you can watch tx hashes appear on the dashboard with zero
real movement.

## Go to testnet (nothing costs real money)

1. In `manca.config.json` set `settlement.x402.mode = "testnet"`.
2. Install the signer (optional dependency, only needed for real modes):
   ```bash
   npm i viem
   ```
3. Fund a **Manca escrow wallet** with Base Sepolia testnet USDC and export its key:
   ```bash
   export X402_PRIVATE_KEY=0x<escrow wallet private key>
   ```
   (Base Sepolia USDC: `0x036CbD53842c5426634e7929541eC2318f3dCF7e`, chain `84532`.)
4. Give sellers an EVM **payout address** (`account.payoutAddress`) so settlement has a destination.
5. Run trades. Each settlement now signs an EIP‚Äë3009 `transferWithAuthorization`,
   posts it to the facilitator (`https://x402.org/facilitator`) `/verify` then
   `/settle`, and stores the returned on‚Äëchain **transaction hash** on the trade.
   The dashboard links each tx to `sepolia.basescan.org`.

## Go to mainnet (REAL money ‚Äî deliberate)

Everything above, plus **all** of:

- `settlement.x402.mode = "mainnet"`, `network = "eip155:8453"`,
  `facilitator = "https://api.cdp.coinbase.com/platform/v2/x402"`,
  `asset` = Base mainnet USDC.
- `settlement.x402.allowMainnet = true` **or** `export X402_ALLOW_MAINNET=1`.

Without the opt‚Äëin flag, the mainnet rail refuses to settle. This is intentional ‚Äî
real funds never move by accident.

## How it maps to Manca's escrow

Manca still runs its own escrow + machine‚Äëverified fulfillment in its ledger. The
rail is invoked at the moment a trade **settles**: it moves the seller's proceeds
from the Manca escrow wallet to the seller's payout address. If the on‚Äëchain
settlement fails, Manca does **not** credit the seller ‚Äî the trade stays `matched`
and can be retried or expired (buyer refunded). Verified against
`coinbase/x402 ‚Üí specs/schemes/exact/scheme_exact_evm.md`.
